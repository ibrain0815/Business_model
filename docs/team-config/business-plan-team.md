# 사업계획서 작성 팀 구성

> AI/기술 스타트업 정부지원사업용 사업계획서 작성을 위한 서브에이전트 팀 구성

## 팀 구조 개요

```
┌─────────────────────────────────────────────────────────┐
│                    CTO Lead (조율자)                      │
│              전체 흐름 관리 및 품질 검증                    │
└─────────────────────┬───────────────────────────────────┘
                      │
        ┌─────────────┴─────────────┐
        │                           │
┌───────┴───────┐           ┌───────┴───────┐
│   리서치 팀    │           │    기획 팀     │
│  (Research)   │    →      │  (Planning)   │
└───────────────┘           └───────────────┘
```

## 1. 리서치 팀 (Research Team)

### 역할
시장 조사, 경쟁사 분석, 기술 동향 파악, 정부지원사업 트렌드 분석

### 서브에이전트 구성

| 에이전트 | 역할 | 담당 업무 |
|---------|------|----------|
| **market-researcher** | 시장 분석가 | 시장 규모, 성장률, TAM/SAM/SOM 분석 |
| **competitor-analyst** | 경쟁사 분석가 | 경쟁사 현황, SWOT, 차별화 포인트 도출 |
| **tech-scout** | 기술 동향 분석가 | AI/기술 트렌드, 기술 로드맵 자료 수집 |
| **policy-researcher** | 정책 리서처 | 정부지원사업 요건, 평가 기준, 가점 요소 분석 |

### 리서치 산출물

1. **시장 분석 보고서** (`docs/research/market-analysis.md`)
   - 시장 규모 및 성장 전망
   - 타겟 고객 세그먼트
   - 시장 진입 기회

2. **경쟁사 분석 보고서** (`docs/research/competitor-analysis.md`)
   - 주요 경쟁사 프로파일
   - 경쟁 구도 분석
   - 차별화 전략 제안

3. **기술 동향 보고서** (`docs/research/tech-trends.md`)
   - 핵심 기술 트렌드
   - 기술 성숙도 분석
   - 기술 적용 로드맵

4. **정부지원사업 가이드** (`docs/research/policy-guide.md`)
   - 적합 지원사업 목록
   - 평가 기준 분석
   - 작성 가이드라인

---

## 2. 기획 팀 (Planning Team)

### 역할
사업계획서 구조 설계, 콘텐츠 작성, 재무 계획, 문서 품질 관리

### 서브에이전트 구성

| 에이전트 | 역할 | 담당 업무 |
|---------|------|----------|
| **business-strategist** | 사업 전략가 | 비전/미션, 사업 모델, 수익 구조 설계 |
| **content-writer** | 콘텐츠 작성자 | 사업계획서 본문 작성, 스토리라인 구성 |
| **financial-planner** | 재무 기획자 | 재무 계획, 손익 분석, 자금 운용 계획 |
| **document-reviewer** | 문서 검토자 | 논리 검증, 일관성 체크, 최종 품질 관리 |

### 기획 산출물

1. **사업 개요** (`docs/plan/executive-summary.md`)
   - 사업 비전 및 미션
   - 핵심 가치 제안
   - 사업 목표

2. **사업 모델** (`docs/plan/business-model.md`)
   - 비즈니스 모델 캔버스
   - 수익 모델
   - 가치 사슬

3. **재무 계획** (`docs/plan/financial-plan.md`)
   - 매출 계획
   - 비용 구조
   - 손익 분기점 분석
   - 자금 조달 및 운용 계획

4. **실행 계획** (`docs/plan/execution-plan.md`)
   - 마일스톤
   - 조직 구성
   - 리스크 관리

## 정부지원사업용 사업계획서 구조

정부지원사업(창업지원, R&D 과제 등) 평가 기준에 맞춘 구조:

### 필수 섹션

| 섹션 | 내용 | 담당 |
|-----|------|------|
| 1. 사업 개요 | 창업 아이템 개요, 대표자 역량 | 기획팀 |
| 2. 문제 정의 | 해결하고자 하는 문제/Pain Point | 리서치팀 + 기획팀 |
| 3. 솔루션 | 제품/서비스 설명, 기술 우위성 | 기획팀 |
| 4. 시장 분석 | TAM/SAM/SOM, 시장 트렌드 | 리서치팀 |
| 5. 경쟁 분석 | 경쟁사 현황, 차별화 전략 | 리서치팀 |
| 6. 비즈니스 모델 | 수익 모델, 가격 전략 | 기획팀 |
| 7. 마케팅 전략 | Go-to-Market 전략 | 기획팀 |
| 8. 재무 계획 | 매출/비용 계획, 자금 운용 | 기획팀 |
| 9. 팀 구성 | 조직도, 핵심 인력 역량 | 기획팀 |
| 10. 실행 로드맵 | 마일스톤, 일정 계획 | 기획팀 |

## 에이전트 호출 방법

### 리서치 팀 실행
```bash
# 전체 리서치 실행
/pdca team research business-plan

# 개별 리서치 실행
Task(market-researcher): "AI 스타트업 시장 규모 및 성장 전망 분석"
Task(competitor-analyst): "국내외 AI 스타트업 경쟁사 분석"
Task(tech-scout): "2024-2025 AI 기술 트렌드 분석"
Task(policy-researcher): "2025년 정부 창업지원사업 분석"
```

### 기획 팀 실행
```bash
# 전체 기획 실행
/pdca team planning business-plan

# 개별 기획 실행
Task(business-strategist): "AI 솔루션 비즈니스 모델 설계"
Task(content-writer): "사업계획서 초안 작성"
Task(financial-planner): "3개년 재무 계획 수립"
Task(document-reviewer): "사업계획서 최종 검토"
```

## PDCA 사이클 적용

| PDCA | 활동 | 담당 |
|------|------|------|
| **Plan** | 사업계획서 구조 설계, 리서치 범위 정의 | CTO Lead |
| **Do** | 리서치 실행, 콘텐츠 작성 | 리서치팀 + 기획팀 |
| **Check** | 문서 검토, 논리 검증, 갭 분석 | document-reviewer |
| **Act** | 피드백 반영, 문서 수정, 최종화 | 전체 팀 |

## 다음 단계

1. `/pdca plan business-plan` - 사업계획서 Plan 문서 생성
2. 리서치 팀 태스크 할당 및 실행
3. 리서치 결과 기반 기획 팀 태스크 실행
4. `/pdca analyze business-plan` - 갭 분석
5. `/pdca report business-plan` - 최종 보고서 생성
